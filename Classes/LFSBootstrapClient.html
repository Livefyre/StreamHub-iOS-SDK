getContentForPage:onSuccess:onFailure:

Get content for page

- (void)getContentForPage:(NSInteger)pageIndex onSuccess:(LFSSuccessBlock)success onFailure:(LFSFailureBlock)failure

Parameters

pageIndex

The page to fetch content for. The pages are numbered from zero. If the page index provided is outside the bounds of what the init data knows about the error callback will convey that failure.

success

Success callback

failure

Failure callback

Discussion

The init data for a collection contains information about that collection’s number of pages and where to fetch the content for those pages.

getUserContentForUser:token:statuses:offset:onSuccess:onFailure:

Fetches the user’s content history

- (void)getUserContentForUser:(NSString *)userId token:(NSString *)userToken statuses:(NSArray *)statuses offset:(NSInteger)offset onSuccess:(LFSSuccessBlock)success onFailure:(LFSFailureBlock)failure

Parameters

userId

The Id of the user whose content is to be fetched.

userToken

(optional) The lftoken of the user whose content is to be fetched. This parameter is required by default unless the network specifies otherwise.

statuses

(optional) array of comment states to return.

offset

(optional) Number of results to skip, defaults to 0. 25 items are returned at a time.

success

Success callback

failure

Failure callback

Discussion

Fetches the user’s content history. For more information see: https://github.com/Livefyre/livefyre-docs/wiki/User-Content-API
