authenticateUserWithToken:forCollection:article:site:network:onSuccess:onFailure:

Check a user’s token against the auth admin.

+ (void)authenticateUserWithToken:(NSString *)userToken forCollection:(NSString *)collectionId article:(NSString *)articleId site:(NSString *)siteId network:(NSString *)networkDomain onSuccess:(void ( ^ ) ( NSDictionary *userData ))success onFailure:(void ( ^ ) ( NSError *error ))failure

Parameters

userToken

The lftoken representing a user.

collectionId

(optional) The Id of the collection to auth against.

articleId

(optional) The Id of the collection’s article.

siteId

(optional) The Id of the article’s site.

networkDomain

The collection’s network as identified by domain, i.e. livefyre.com.

success

Callback called with a dictionary after the user data has been retrieved.

failure

Callback called with an error after a failure to retrieve data.

Return Value

void

Discussion

It is necessary to provide either a collectionId or a siteId combined with an articleId.
